Linux 防火墙流程控制与洪水攻击识别

实验环境： AWS EC2 (Amazon Linux / Ubuntu) | 工具： iptables, ab, ping

⚠️ 注意： 实验前请确保 AWS 安全组（Security Group）已放行所有流量，否则系统内部规则无法被外部感知。

1. 基础环境安装

首先安装并初始化 iptables 服务：

sudo yum install iptables-services -y
sudo systemctl start iptables
sudo iptables -F

2. 端口与 ICMP 控制实验

A. 80 端口开关测试

# 关闭 80 端口访问
sudo iptables -A INPUT -p tcp --dport 80 -j DROP

# 恢复 80 端口访问
sudo iptables -D INPUT -p tcp --dport 80 -j DROP

B. Ping (ICMP) 控制

# 禁止外部 Ping 我
sudo iptables -A INPUT -p icmp --icmp-type echo-request -j DROP

3. 出站流量拦截 (特定 IP)

假设禁止服务器访问目标 IP 110.242.68.66：

sudo iptables -A OUTPUT -d 110.242.68.66 -j DROP

4. 洪水/异常流量识别与限流（防护配置）

A. ICMP 异常流量限流与记录

# 1. 限流放行正常包
sudo iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s --limit-burst 5 -j ACCEPT
# 2. 记录超限事件（对日志本身也做限速，避免日志被刷爆）
sudo iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 2/s --limit-burst 20 -j LOG --log-prefix "FLOOD-LOG: "
# 3. 拦截超限流量
sudo iptables -A INPUT -p icmp --icmp-type echo-request -j DROP

验证方式（安全）： 使用少量 ping 验证规则生效，并通过计数器与日志前缀证明限流与拦截链路工作。

B. HTTP 并发异常识别与拦截

# 限制单个 IP 并发连接数 > 10 则拦截
sudo iptables -A INPUT -p tcp --syn --dport 80 -m connlimit --connlimit-above 10 -m limit --limit 5/s --limit-burst 50 -j LOG --log-prefix "HTTP-FLOOD: "
sudo iptables -A INPUT -p tcp --syn --dport 80 -m connlimit --connlimit-above 10 -j DROP

验证方式（安全）： 用浏览器或少量 curl 访问站点，配合计数器与日志前缀作为证据。

5. 规则查看与清理

目标 命令
查看实时计数器 sudo iptables -nvL --line-numbers
查看攻击/超限日志 sudo dmesg | grep -E "FLOOD|HTTP-FLOOD"
清空所有规则 sudo iptables -F
